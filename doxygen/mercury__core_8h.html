HG_EXPORT hg_return_t HG_Core_registered_disable_response ( hg_class_t *  hg_class,
hg_id_t  id,
hg_bool_t  disable 
)

Disable response for a given RPC ID.

This allows an origin process to send an RPC to a target without waiting for a response. The RPC completes locally and the callback on the origin is therefore pushed to the completion queue once the RPC send is completed. By default, all RPCs expect a response to be sent back.

Parameters
hg_class[IN] pointer to HG class
id[IN] registered function ID
disable[IN] boolean (HG_TRUE to disable HG_FALSE to re-enable)
Returns
HG_SUCCESS or corresponding HG error code

◆ HG_Core_forward()

HG_EXPORT hg_return_t HG_Core_forward ( hg_handle_t  handle,
hg_cb_t  callback,
void *  arg,
hg_bulk_t  extra_in_handle,
hg_size_t  size_to_send 
)

Forward a call using an existing HG handle.

Input and output buffers can be queried from the handle to serialize/deserialize parameters. Additionally, a bulk handle can be passed if the size of the input is larger than the queried input buffer size. After completion, the handle must be freed using HG_Core_destroy(), user callback is placed into a completion queue and can be triggered using HG_Core_trigger().

Parameters
handle[IN] HG handle
callback[IN] pointer to function callback
arg[IN] pointer to data passed to callback
extra_in_handle[IN] bulk handle to extra input buffer
size_to_send[IN] size of request to transmit
Returns
HG_SUCCESS or corresponding HG error code
