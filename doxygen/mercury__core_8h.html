◆ HG_Core_set_more_data_callback()

HG_PUBLIC hg_return_t HG_Core_set_more_data_callback ( struct hg_core_class *  hg_core_class,
hg_return_t(*)(hg_core_handle_t, hg_op_t, hg_return_t(*done_callback)(hg_core_handle_t))  more_data_acquire_callback,
void(*)(hg_core_handle_t)  more_data_release_callback 
)

Set callback that will be triggered when additional data needs to be transferred and HG_Core_set_more_data() has been called, usually when the eager message size is exceeded.

This allows upper layers to manually transfer data using bulk transfers for example. The done_callback argument allows the upper layer to notify back once the data has been successfully acquired. The release callback allows the upper layer to release resources that were allocated when acquiring the data.

Parameters
hg_core_class[IN] pointer to HG core class
more_data_acquire_callback[IN] pointer to acquire function callback
more_data_release_callback[IN] pointer to release function callback
Returns
HG_SUCCESS or corresponding HG error code

◆ HG_Core_context_post()

HG_PUBLIC hg_return_t HG_Core_context_post ( hg_core_context_t *  context)

Post requests associated to context in order to receive incoming RPCs.

Requests are automatically re-posted after completion until the context is destroyed. Additionally a callback can be triggered on HG handle creation. This allows upper layers to instantiate data that needs to be attached to a handle. Number of requests that are posted can be controlled through HG init info.

Parameters

Returns
